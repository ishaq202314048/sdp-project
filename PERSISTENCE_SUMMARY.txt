HOW LOGIN STORAGE WORKS
─────────────────────────────────────────────────────────────────────────

1. User submits login form (email + password)
2. API validates credentials against User table
3. If valid:
   ✓ JWT token generated
   ✓ LoginSession record created (with IP, user-agent, timestamp)
   ✓ User session available for audit logs
4. Next logins by same user create additional records

Example session entry:
  {
    userId: "3cc051dc-...",
    email: "a@gmail.com",
    userType: "soldier",
    ipAddress: "192.168.x.x",
    userAgent: "Mozilla/5.0...",
    loginAt: "2026-01-26T10:15:30.000Z"
  }
